# docs-agent — Автоматическое ведение документации

## Что это

`docs-agent` — инструмент, который следит за изменениями в коде и автоматически обновляет документацию проекта. Разработчик коммитит код — агент смотрит что изменилось и актуализирует README и wiki, не отвлекая команду.

## Как это работает

```
Разработчик делает commit / push
        │
        ▼
CI запускает docs-agent
        │
        ├─ Первый раз в проекте?
        │   └─ Генерирует документацию с нуля по шаблону
        │
        └─ Проект уже настроен?
            ├─ Нет релевантных изменений → пропускает
            └─ Есть изменения в коде → обновляет нужные секции
                        │
                        ▼
             Создаёт PR с обновлёнными доками
             (или коммитит напрямую — настраивается)
```

Агент обновляет только те секции документа, которых коснулись изменения. Разделы, которые написаны вручную, не трогаются.

## Что обновляется

- **README.md** — описание сервиса, как запустить, основные команды
- **Wiki-страницы:**
  - `architecture.md` — компоненты системы, схема взаимодействия сервисов, технические решения
  - `api-contracts.md` — эндпоинты, форматы запросов/ответов, коды ошибок, аутентификация
  - `development-guide.md` — настройка окружения, запуск тестов, стиль кода, правила ревью
  - `operations.md` — runbooks, мониторинг, процедуры деплоя, действия при инцидентах
  - `data-model.md` — доменные сущности, бизнес-правила, связи между объектами
  - `DB-schema.md` — схема базы данных, таблицы, индексы, миграции
  - `integrations.md` — внешние сервисы и API с которыми работает проект
  - `security.md` — модель доступа, обработка чувствительных данных, требования безопасности
  - `troubleshooting.md` — типичные проблемы и способы их диагностики
  - `adr.md` — Architecture Decision Records, история архитектурных решений

Набор документов настраивается под каждый проект.

## Как подключить проект

Одна команда в корне любого репозитория:

```bash
pip install docs-agent
docs-agent install
```

Задаёт 2 вопроса (какой AI-провайдер, автокоммит или PR) и создаёт:
- `.docs-agent/docs-agent.yml` — конфиг
- Таргеты в `Makefile`: `make docs`, `make docs-sync`
- CI workflow (GitHub Actions или GitLab CI)

После этого — `make docs` для первой генерации, дальше всё автоматически.

## Поддерживаемые AI-провайдеры

| Провайдер | Когда использовать |
|-----------|-------------------|
| **Claude (Anthropic)** | Основной вариант, лучшее качество текста |
| **OpenAI (GPT-4o)** | Альтернатива если есть ключи OpenAI |
| **Ollama** | Локальная модель, без внешних API — для закрытых проектов |

Провайдер задаётся в конфиге, переключается без изменения кода.

## Управление всеми проектами

После подключения нескольких проектов — централизованный обзор:

```bash
docs-agent dashboard   # таблица в терминале
docs-agent report      # HTML-страница со статусами
```

Показывает: какие проекты подключены, когда последний раз обновлялась документация, есть ли устаревшие.

## Для команды — что нужно сделать

1. Запустить `docs-agent install` в своём репозитории
2. Добавить API-ключ в секреты CI (`ANTHROPIC_API_KEY` или `OPENAI_API_KEY`)
3. Запустить `make docs` один раз для первичной генерации
4. Дальше всё работает само при каждом push

## Шаблоны документов

Шаблоны хранятся централизованно в пакете. Если нужен кастомный стиль — можно положить свои шаблоны в `.docs-agent/templates/` внутри репозитория. Это позволяет соблюдать единый стиль документации по всей компании, обновляя шаблоны в одном месте.
